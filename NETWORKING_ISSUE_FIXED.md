# E2E Performance Tests - Networking Issue Resolution

## Summary

Successfully investigated and resolved all networking and configuration issues preventing the E2E performance tests from running.

## Issues Found and Fixed

### 1. IPv6 Connectivity Issue ✅ FIXED
**Problem**: Services running in Docker containers were not responding to IPv6 (::1) connections from the test environment.

**Root Cause**: .NET HttpClient defaults to IPv6 when using "localhost" as the hostname, but the Docker services were not listening on IPv6.

**Solution**: Changed all service URLs from `localhost` to `127.0.0.1` to force IPv4 connections.

**Files Modified**:
- `E2E.PerformanceTests/Infrastructure/TestConfig.cs`
  - Changed API Gateway URL: `http://localhost:8080` → `http://127.0.0.1:8080`
  - Changed TripService URL: `http://localhost:5002` → `http://127.0.0.1:5002`
  - Changed DriverService URL: `http://localhost:5003` → `http://127.0.0.1:5003`
  - Changed Redis connection: `localhost:6379` → `127.0.0.1:6379`
  - Changed PostgreSQL host: `localhost` → `127.0.0.1`
  - Changed RabbitMQ host: `localhost:5672` → `127.0.0.1:5672`

### 2. Redis Admin Commands Issue ✅ FIXED
**Problem**: Redis `FLUSHDB` command failed because admin mode was not enabled.

**Root Cause**: Performance tests were trying to use `FlushDatabaseAsync()` which requires admin privileges.

**Solution**: Replaced database flush with selective key deletion using patterns.

**Files Modified**:
- `E2E.PerformanceTests/Infrastructure/RedisHelper.cs`
  - Modified `CleanupAsync()` method to delete specific keys instead of flushing entire database
  - Added error handling for non-critical cleanup failures

### 3. JWT Token Configuration Mismatch ✅ FIXED
**Problem**: All API requests returned 401 Unauthorized errors.

**Root Cause**: JWT tokens generated by tests used different signing key, issuer, and audience than the services expected.

**Configuration Mismatch**:
- Test Helper: Secret="this-is-a-very-secure...", Issuer="uit-go", Audience="uit-go-users"
- Services: Secret="UltimateSuperSecretKeyForJwtTokenGeneration123", Issuer="UserService", Audience="UserServiceClients"

**Solution**: Updated JWT token helper to match service configuration.

**Files Modified**:
- `E2E.PerformanceTests/Infrastructure/JwtTokenHelper.cs`
  - Changed SecretKey to match services
  - Changed Issuer to "UserService"
  - Changed Audience to "UserServiceClients"

### 4. Missing Request Model ✅ FIXED
**Problem**: API requests returned 415 Unsupported Media Type errors.

**Root Cause**: The `CreateTripRequest` record was not defined in the controller, causing model binding to fail.

**Solution**: Added the missing request model definition.

**Files Modified**:
- `TripService/TripService.Api/Controllers/TripsController.cs`
  - Added: `public record CreateTripRequest(double PickupLat, double PickupLng, double DropoffLat, double DropoffLng);`

### 5. HTTP Content-Type Issue ✅ FIXED
**Problem**: Even after adding the request model, 415 errors persisted.

**Root Cause**: `StringContent` was created without specifying the media type parameter, even though the header was set separately.

**Solution**: Properly instantiate `StringContent` with encoding and media type parameters.

**Files Modified**:
- `E2E.PerformanceTests/Workloads/WorkloadA_TripCreation.cs`
  - Changed from: `.WithHeader("Content-Type", "application/json").WithBody(new StringContent(json))`
  - Changed to: `.WithBody(new StringContent(json, System.Text.Encoding.UTF8, "application/json"))`

### 6. Non-Interactive Console Issues ✅ FIXED
**Problem**: Tests failed when run in non-interactive mode due to `Console.Clear()` and `Console.ReadKey()`.

**Solution**: Added error handling and environment checks for console operations.

**Files Modified**:
- `E2E.PerformanceTests/Program.cs`
  - Wrapped `Console.Clear()` in try-catch
  - Made `Console.ReadKey()` conditional on `Environment.UserInteractive`

### 7. Statistics Parsing Issue ✅ FIXED
**Problem**: "Index was outside the bounds of the array" exception when accessing test statistics.

**Root Cause**: No null/bounds checking before accessing stats arrays.

**Solution**: Added proper validation and graceful handling.

**Files Modified**:
- `E2E.PerformanceTests/Workloads/WorkloadA_TripCreation.cs`
  - Added bounds checking for `ScenarioStats` and `StepStats` arrays
  - Added fallback message when step stats are not available
